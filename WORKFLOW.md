# 小贝壳项目工作流程

## 1. 目的
明确项目开发流程，确保高效、透明的协作

## 2. 核心流程
### 步骤一：制定开发计划
- 我负责：
  - 基于需求分析制定详细的开发计划
  - 包括任务分解、时间表、预期结果
  - 评估技术难度和风险

- 交付成果：
  - 开发计划文档（如DEV_PLAN.md）
  - 任务列表和优先级
  - 预计完成时间

### 步骤二：等待确认
- 您负责：
  - 阅读开发计划
  - 确认计划的合理性和可行性
  - 提出修改意见或建议
  - 最终确认或拒绝计划

### 步骤三：执行开发
- 我负责：
  - 严格按照确认后的计划执行
  - 定期汇报进度
  - 遇到问题及时沟通
  - 完成后提交成果

## 3. 沟通机制
- 使用飞书作为主要沟通工具
- 重要决策必须书面确认
- 每周至少一次进度汇报

## 4. 文件格式规范
- 开发计划文件格式：DEV_PLAN_YYYYMMDD.md
- 进度汇报格式：PROGRESS_YYYYMMDD.md
- 需求变更格式：CHANGE_REQUEST_YYYYMMDD.md
